Song Data Audio Commands

first nibble is command
second nibble is command parameter
some commands have trailing data
*maybe features

0x set Base Temporal Unit size
1x set BTUs until next action
2x channel(s) trigger note #
   each bit represents channels pu1..noise
   amount of data correlates to channels enabled
3x trigger percussion macro
4x pu1 set envelope
5x pu2 set envelope
6x tri set BTU length
7x noi set envelope
8x pulses set duty cycle (pu1/pu2)
Ax set loop1 point with count x
Bx set loop2 point with count x
*Cx set pitch down speed
*Dx set pitch up speed
Ex halt channels (song-only)
   each bit represents channels pu1..noise
   immediate silence + clear channel counters
Fx song controls
   0 loop back to beginning
   3 mend1 set begin pointer
   4 mend1 set end pointer (or jump to mend1 next if already set)
   5 mend1 set next pointer and jump to begin pointer
   6 mend2 set begin pointer
   7 mend2 set end pointer (or jump to mend2 next if already set)
   8 mend2 set next pointer and jump to begin pointer
   A loop back to loop1 point	
   B loop back to loop2 point
   F end song


SFX Melodic Sequence Data
always uses pu2 channel
first byte is envelope id then
2 byte pairs: note id (negative means end) and frames until next note action
